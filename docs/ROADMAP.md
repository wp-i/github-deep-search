# Roadmap

GitHub Deep Search is optimized for one job: help builders quickly discover whether similar open-source projects already exist, how close they are, and what can be reused.

## Near Term

- Improve first-run onboarding and example queries.
- Add more actionable error messages for missing keys, rate limits, and provider failures.
- Expand rendered Web regression coverage for mobile and narrow desktop layouts.
- Improve report structure for direct reuse, reference-only, and adjacent-project candidates.

## Search Quality

- Improve `SearchSpec` parsing for ambiguous product ideas.
- Add stronger evidence coverage scoring without hard-coded product-domain word packs.
- Better distinguish catalogs, templates, prototypes, and production-ready projects.
- Add saved live-eval cases that contributors can run with their own credentials.

## Distribution

- Keep one-command startup reliable across Windows, macOS, and Linux.
- Maintain Docker Compose as a second fast path.
- Keep real-result screenshots fresh, with query, capture date, and usage noted.
- Prepare launch posts and reproducible examples for developer communities.

## Possible Later Work

- Browser extension or CLI integration for searching while reading product specs.
- Export report bundles with evidence snippets.
- Compare multiple candidate project ideas in one run.
- Optional hosted mode for teams that do repeated open-source discovery.
